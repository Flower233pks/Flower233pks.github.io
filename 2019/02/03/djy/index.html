前言

HLPP(Highest Label Preflow Push)HLPP(Highest~Label~Preflow~Push)HLPP(Highest Label Preflow Push)最高标签预流推进算法是处理网络最大流里两种常用方法——增广路&预流推进中，预流推进算法的一种。据传由tarjantarjantarjan发明怎么又是他 ，并被其他科学家证明了其复杂度是紧却的O(n2m)O(n^2 \sqrt m)O(n2m​)。在随机数据中不逊色于普通的增广路算法，而在精心构造的数据中无法被卡，所以是一种可以替代DinicDinicDinic的方法（随我怎么说，代码又长又难调，所以还是DinicDinicDinic好啊TAT\rm{TAT}TAT）

但无论怎样，wikiwikiwiki里面已经承认HLPPHLPPHLPP是现在最优秀的网络流算法了。

那么预流推进这个大门类里面，思想都差不多。大抵上就是我们对每个点记录超额流(Extra FlowExtra~FlowExtra Flow) ，即允许流在非源点暂时存储，并伺机将超额流推送出去。不可推送的，就会流回源点。那么最终答案显然存储在Extra[T]Extra[T]Extra[T]里面。

但同时这也有一个问题，就是会出现两个点相互推送不停的情况。为了防止这样，我们采用最高标号的策略，给每个点一个高度，对于一个点uuu以及它的伴点集合{v}\{v\}{v}，当且仅当hu=hv+1h_u = h_v + 1hu​=hv​+1 时才可以推送流。并且我们对于源点SSS，设置hS=Nh_S = NhS​=N，并对于SSS实行无限制推送。那么最后的答案就保存在Extra[T]Extra[T]Extra[T]里面 。

但有时，我们发现有个点是”谷“，即周围点的高度都比它高，但是它有超额流。那么我们此时考虑拔高它的高度，即**重贴标签(relabelrelabelrelabel)**操作。
